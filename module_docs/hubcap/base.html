class hubcap.base.Issues(repo: str | ~github.Repository.Repository, get_objs=<function Repository.get_issues>, *, objs_to_items='number', data_of_obj=<function identity>, get_objs_kwargs=(('state', 'open'), ))[source]

Mapping interface to repository issues.

Parameters:

  • repo – The repository to get issues from

  • get_objs – The function to get the issues

  • objs_to_items – A function to get the key-value pairs from the issue objects

  • data_of_obj – A function to get the data from the issue object. Default is attrgetter(‘body’). Use identity to get the whole issue object.

  • get_objs_kwargs – The keyword arguments to pass to the get_objs function

The default of data_of_obj is IssueContents, which will give a mapping interface to the body and comments of the issue.
